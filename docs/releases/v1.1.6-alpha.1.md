# Alpha 1.1.6

Language-selection, Launchpad Rosetta localization, and Russian-productization release for the Alpha 1.1 line.

## Included

- Adds a first-run and signed-out language gate ahead of auth/setup, with automatic system-locale selection, a dedicated full-screen selector, and a multilingual animated background so interface language is chosen before the rest of OOBE starts.
- Adds per-account interface language persistence with a new `user_preferences.locale` schema field plus Settings > Interface controls, and returns the saved locale on setup completion, login, passkey login, and `/api/v1/me` so signed-in sessions consistently reopen in the user’s chosen language.
- Replaces the earlier browser-only locale shim with native Launchpad Rosetta-backed `.lpr` loading by vendoring the upstream runtime sources and Go binding, parsing `web/locales/*.lpr` on the server, and serving browser-ready bundles from `/api/v1/public/i18n/{locale}`.
- Ships a thorough Russian locale catalog in Launchpad Rosetta format and wires Russian copy through the main product chrome, including auth, OOBE, settings, admin, mail, passkey, authenticator-app, recovery-code, and recovery-email MFA flows.
- Improves Russian localization quality so UI text reads naturally for native speakers instead of as a literal English calque, and fixes the DOM relocalization edge case where already translated Russian strings containing the `Despatch` brand could collapse back to a partial English-looking fragment.
- Makes external-account-only OOBE more honest about infrastructure assumptions by removing the admin recovery-email requirement and UI when setup is intentionally configured for Gmail/Libero/other external providers without a local mail stack.
