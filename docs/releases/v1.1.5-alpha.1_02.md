# Alpha 1.1.5_02

Local-mail account recovery and indexed mailbox resilience hotfix for the Alpha 1.1.5 line.

## Included

- Fixes indexed mail falling back to `Live mailbox · indexed view unavailable` when aggregate mailbox-name resolution cannot be refreshed from the live server, by preserving safe role-aware mailbox fallbacks like `INBOX`, `Sent`, `Trash`, `Archive`, and `Junk`.
- Fixes local SQL-backed installs that could show live mail but no indexed account in Settings, preventing indexed message opens and account-aware mail actions from resolving cleanly until a manual account existed.
- Limits automatic account repair to the indexed accounts surface for local-mail instances so the missing-account recovery no longer disturbs unrelated health or draft-send flows.
- Replaces the hand-maintained startup migration list with automatic sorted migration discovery so newer checked-in schema files are not silently skipped at boot.
