# Alpha 1.1.2_04

Verified Launchpad installer release for the Alpha 1.1.2 line, now with restored native parity for Dovecot SQL autodetection.

## Included

- Keeps the verified Launchpad installer and release-provenance pipeline in place for universal and architecture-specific installer artifacts.
- Makes the universal Launchpad installer the primary installation entrypoint and keeps manual verification support through Launchpad proof and attestation metadata.
- Restores native installer parity for Dovecot SQL mailbox mapping by using Launchpad `v1.2.3` semantic extraction instead of hardcoded defaults.
- Auto-detects the Dovecot SQL auth table, login column, password-hash column, active column, and mail-location column from `password_query` and `user_query`.
- Updates the installer form and review flow so the inferred Dovecot SQL mapping is visible before installation.
- Fixes native SQL driver resolution so the configured installer value is respected consistently.
- Picks up Launchpad `v1.2.3` runtime fixes for fullscreen installer rendering, the public `enable service` / `start service` statement path, and optional Launchpad-owned JSON state loading.
- Hardens installer metadata reads so an empty or interrupted `/var/lib/despatch/launchpad-install.json` no longer crashes the Launchpad lifecycle with a bare JSON parse error.
- Removes extract-stage dry-run assertions that could fail before the real install path reached native preflight and host wiring.
- Moves Despatch unit start/stop state back into the native service declarations so the installer no longer depends on the fragile manual service-control block.
- Ignores the project-local Launchpad extension index cache so stale `.launchpad` runtime state does not get carried forward in source publishes.
