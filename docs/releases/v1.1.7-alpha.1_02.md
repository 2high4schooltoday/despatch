# Alpha 1.1.7_02

Language-persistence, stale-asset hardening, and outbound-operations onboarding hotfix release for the Alpha 1.1.7 line.

## Included

- Fixes signed-out language persistence more thoroughly by storing the guest locale in both durable browser storage and a dedicated cookie, so the OOBE language choice survives stricter embedded-browser and reinstall scenarios.
- Stops stale web assets from pinning the UI to old behavior by rendering versioned `app.js` / `i18n.js` / `styles.css` URLs into `index.html`, serving the web root from the correct runtime path, and disabling caching on HTML, JS, CSS, setup-status, auth-session, and public-locale bootstrap responses.
- Fixes the frontend runtime regression where outbound state parsing could throw `deepClone is not defined`, restoring the affected `Outbound` flows and centralizing safe value cloning.
- Adds a much more explicit inline guide inside `Outbound` and `Reply Ops`, including beginner-first checklists, field explanations, plain-language glossary sections, and step-by-step real-world examples for multi-mailbox external-account operations such as many Libero source accounts feeding one collector desk.
- Ships full Russian localization coverage for the new onboarding guide content so the new helper surfaces stay consistent with the rest of the Russian product experience.
