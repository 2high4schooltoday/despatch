# Alpha 1.1.4

Funnel-routing, provider-integration, conversation-threading, and installer-correctness release for the Alpha 1.1 line.

## Included

- Adds provider-aware external-account workflows on top of the new local/external/hybrid instance model, including username-first admin bootstrap for external-account installs and a cleaner OOBE path for setups that do not own a local mail domain.
- Adds first-class provider presets and capability messaging for Libero and Gmail, including guided app-password help, provider-aware validation, and assisted forwarding flows that keep multi-account setup honest about what Despatch can automate versus guide.
- Adds reply funnels as a high-level workflow instead of making operators hand-build dozens of rules, with collector/source account mapping, managed redirect-rule stamping, unified funnel inbox access, unlimited source-account scale, and sender selection that follows the active funnel/thread context.
- Improves mail conversations and historical repair logic so threading uses `Message-ID`, `In-Reply-To`, `References`, Outlook conversation hints, participant overlap, and guarded fallback heuristics instead of mostly trusting raw subject matching.
- Improves authentication and operator ergonomics with clearer passkey-origin diagnostics, longer-lived sessions plus visible-tab keepalive behavior, and better fallback behavior when connected sender identities are not ready yet.
- Refreshes the Launchpad-packaged installer to Launchpad `v1.3.2`, which fixes authored-stage progress metadata, preserves authored stage ordering in the installer rail, switches progress into determinate mode once stage events match, and restores the progress strip styling to the surrounding Launchpad surface.
