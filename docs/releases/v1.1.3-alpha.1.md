# Alpha 1.1.3

External-account setup, admin mailbox lifecycle, MFA fallback, and mail-workspace reliability release for the Alpha 1.1 line.

## Included

- Adds three first-run instance modes: `local_stack`, `external_accounts`, and `hybrid`, with setup screens, summaries, and validation that only ask for the mail-domain and admin-email semantics required by the chosen mode.
- Decouples Despatch sign-in identity from local mailbox identity in external-accounts mode, including username-based admin bootstrap, username-oriented login/reset labels, and registration lockout when self-service signup does not make sense.
- Documents the new instance-mode and external-account architecture in `docs/architecture/instance-modes-and-external-accounts.md` so the product model, capability split, and future provider work have a checked-in design reference.
- Adds admin-side user creation from the Users surface, including recovery-email capture, SQL provisioning, PAM mailbox-password verification, accepted mailbox-login persistence, audit logging, and immediate activation instead of waiting for a registration round-trip.
- Adds admin mailbox management for individual users: list, create, and delete folders directly against the stored mailbox credentials, move messages to Trash before destructive folder deletion, and keep mailbox-role mappings and indexed state consistent afterwards.
- Stops stale indexed folders from leaking back into live mailbox lists when Dovecot or the connected account returns the authoritative folder set.
- Adds MFA fallback over recovery-email verification codes, with send/verify APIs, expiry and attempt limits, remember-device support, and audit entries.
- Adds “Approve From Another Device” MFA flows, including request/cancel/status APIs, pending-approval listing and decision APIs for already-authenticated devices, browser polling, and in-app approval prompts on trusted sessions.
- Improves passkey/auth-capability diagnostics by surfacing request-origin, allowed-origin, and RP-ID mismatch details instead of collapsing those failures into generic unavailability messaging.
- Extends default session lifetime to 8 hours idle and 7 days absolute across runtime config, `.env.example`, and the Launchpad installer environment staging.
- Improves compose and mailbox ergonomics in mixed or external-account setups by allowing subjectless send, showing a direct “Connect a mail account before sending” fallback, and keeping optimistic Sent rows visible while indexed refresh lags behind a successful send.
- Keeps session-level sender profiles valid even before a primary local mailbox exists, while still preferring connected account senders once they are available.
- Adds mail sync repairs for header-based threading drift, including one-time repair bookkeeping, sync-time detection when references point across thread boundaries, and touched-thread repair updates instead of waiting for a full rebuild.
- Hardens mailbox preview generation so HTML, CSS, style/script blocks, and inline `@font-face` or other layout fragments cannot leak into message-list previews from stored snippets, HTML-only mail, or frontend fallback rendering.
- Tidies the mail command bar so action buttons, search, indexed/sync status, and overflow controls share the same height, typography, and alignment.
- Updates the Launchpad packaging workflow to build and package the installer against Launchpad `v1.3.0`.
- Makes the GitHub release workflow apply the checked-in markdown release notes file when one exists, so published release pages stay aligned with the release scope captured in the repository.
