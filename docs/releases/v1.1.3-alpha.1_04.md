# Alpha 1.1.3_04

Installer corrective release for the Alpha 1.1.3 line, focused on stopping public-hostname drift back to the apex domain and making proxy/TLS host mismatches fail fast during install.

## Included

- Fixes the Go update worker to honor `INSTALL_DEPLOY_MODE` when deriving public reset URLs and related host-facing values, while still accepting legacy `DEPLOY_MODE` for backward compatibility.
- Fixes the Rust update worker with the same deploy-mode handling so both updater paths preserve the configured proxy hostname such as `mail.2h4s2d.ru`.
- Writes both `DEPLOY_MODE` and `INSTALL_DEPLOY_MODE` from the Launchpad installer to keep installed environments compatible with both current and older updater logic.
- Clarifies the Launchpad installer UI by separating `Mailbox domain` from `Public web hostname`, reducing the chance of treating the apex mailbox domain as the browser host in reverse-proxy installs.
- Tightens install-time diagnostics so proxy routing and TLS checks fail the install when the configured public hostname is not actually what the local proxy endpoint serves.
- Keeps the earlier exact-host LetsEncrypt path checks and Launchpad `v1.3.1` packaging fixes in place.
