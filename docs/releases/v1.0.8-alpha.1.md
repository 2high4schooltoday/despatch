# Alpha 1.0.8

Mail-reader fidelity and thread-UX release for v1, adding safe HTML rendering without breaking plain-text workflows.

## Included

- Adds HTML-first message rendering in the v1 reader with a per-message `HTML`/`Plain` toggle and sandboxed iframe isolation.
- Extends v1 message payloads with `body_html` and inline attachment metadata (`inline`, `content_id`) while preserving existing fields.
- Implements robust MIME parsing for plain+HTML alternatives, HTML-only fallback, and CID-backed inline media retrieval parity.
- Adds authenticated v1 remote-image proxying (`/api/v1/messages/{id}/remote-image`) with SSRF protections, image-only enforcement, and bounded streaming.
- Improves conversation scanning with thread-aware summaries (`thread_id`) and preview snippets, plus expanded API/UI regression coverage.
