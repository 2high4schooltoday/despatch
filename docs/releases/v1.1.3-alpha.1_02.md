# Alpha 1.1.3_02

Installer reliability release for the Alpha 1.1.3 line, focused on correcting Launchpad-native progress tracking, exact-host TLS behavior, and stale Python-facing installer residue.

## Included

- Splits the native Launchpad install flow into smaller execution stages so the official Launchpad progress surface can track real stage movement instead of sitting inside a few oversized opaque phases.
- Adds explicit native stage messages around long-running host work such as dependency setup, runtime installation, service wiring, proxy reconciliation, firewall updates, artifact verification, and final diagnostics.
- Keeps determinate progress where Launchpad can emit grounded updates natively, especially download, extract, and service-wait actions, instead of relying on misleading wrapper-side progress reconstruction.
- Forces LetsEncrypt certificate and key validation to match the selected proxy hostname exactly, preventing `mail.2h4s2d.ru` from silently reusing the apex-domain certificate paths.
- Removes the strict install-time dependency on the app's broader readiness aggregate so installation no longer fails just because non-core readiness signals are still settling.
- Stops advertising stale Python entrypoints for the supported installer path, updates the remaining root hint to use `./despatch`, and ignores tracked Python bytecode artifacts going forward.
