# Alpha 1.0.6_05

Password reset delivery reliability hardening and workspace hygiene release.

## Included

- Enforces SMTP-backed delivery for public password reset flows (log-only sender is no longer treated as ready for public reset).
- Adds config validation to prevent `PASSWORD_RESET_SENDER=log` when `PASSWORD_RESET_PUBLIC_ENABLED=true`.
- Improves reset lookup to accept either account email or recovery email while preserving non-enumeration behavior.
- Adds regression coverage across config, sender, service, and API reset paths.
- Adds recursive `.DS_Store` ignore patterns to keep Finder metadata out of Git across all folders.
- Fixes updater write-probe permission diagnostics with explicit ownership/mode repair commands, including parent directory requirements.
- Improves installer remediation output for updater request/status directory permission checks.
