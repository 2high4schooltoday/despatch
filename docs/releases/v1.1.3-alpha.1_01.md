# Alpha 1.1.3_01

Installer hotfix release for the Alpha 1.1.3 line, focused on making Launchpad install and uninstall failures explicit and recoverable on real mail hosts.

## Included

- Removes stale legacy `mailclient` Nginx and Apache site files, including `mailclient-ssl.conf`, before Despatch manages current proxy wiring or uninstall cleanup.
- Prevents Apache teardown from failing on uninstall when leftover legacy SSL vhosts reference directives such as `SSLEngine` without the corresponding module being active.
- Adds install-time validation for local IMAP availability so hosts missing Dovecot listeners fail early with a direct prerequisite message instead of a generic readiness error.
- Adds install-time validation for local SMTP availability so hosts missing Postfix or another local SMTP listener fail before the app reaches the opaque `/health/ready` verification step.
- Adds an explicit PAM-mode validation for a local relay on `127.0.0.1:25`, matching the current password-reset sender readiness contract used during startup and installer verification.
- Keeps the existing `Alpha 1.1.3` feature payload unchanged while tightening Launchpad lifecycle behavior around legacy host state and mail-stack prerequisite detection.
