# Alpha 1.1.7_03

Packaged-Rosetta runtime hotfix for the Alpha 1.1.7 line.

## Included

- Fixes the real cause of Russian language selection failing in installed builds: the web locale endpoint could return `404` because the Launchpad Rosetta native library was not present in release payloads, leaving the server-side `.lpr` runtime uninitialized.
- Teaches the server-side web i18n runtime to auto-discover the Rosetta native library from the installed application layout before loading `web/locales/*.lpr`, including safer fallback behavior when `LP_I18N_LIBRARY_PATH` is set incorrectly.
- Ships the Launchpad Rosetta native library inside Linux release archives so installed Despatch instances can actually serve `/api/v1/public/i18n/ru` and other future Rosetta-backed locales.
- Extends update/apply/rollback payload handling so the Rosetta native library is preserved across in-app updates instead of being dropped after the next upgrade.
- Adds regression coverage for packaged-runtime locale loading and for update payload application with the native library present.
